# Contributing

This project welcomes careful improvements.

Good contributions include:

- clearer decision rules,
- better case studies,
- safer response patterns,
- reviewer checklists,
- multilingual examples,
- and empirical evaluation ideas.

Please avoid adding operational details that could enable harm.

When adding a case study, include:

- user request,
- risk level,
- good response pattern,
- bad response pattern,
- reasoning summary.
